## Best Practices

- Always use `table-responsive` wrapper for horizontal scrolling on small screens
- Include proper semantic HTML with `th` elements and `scope` attributes
- Add `role="table"` and `aria-label` for screen reader accessibility
- Use `caption` element to describe table purpose
- Combine table classes for enhanced functionality (hover, striped)
- For mobile, use `table-stacked` with `data-label` attributes

## Accessibility Notes

- Use semantic table markup (`table`, `thead`, `tbody`, `th`, `td`)
- Add `scope="col"` to column headers and `scope="row"` to row headers
- Include `caption` or `aria-label` to describe table purpose
- Ensure sufficient color contrast for table content
- Use `data-label` attributes for stacked mobile layout
- Maintain keyboard navigation and focus management
- Test with screen readers for proper table navigation
- Avoid using tables for layout purposes
